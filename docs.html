devtemDesign Documentation

One configuration schema. Infinite animated synthesis. devtemDesign is a lightweight, zero-configuration animated page engine. Stop building standard UI boilerplate — let the engine paint the viewport with dynamic templates, glassmorphism, ambient motion, and a reactive component schema.

Version: 0.0.2  |  License: MIT  |  Live Demo: design.devtem.org

Quick Start

Add the CDN links to your HTML and call the engine with your content.

<!DOCTYPE html>
<html>
<head>
  <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/devtemdesign@0.0.2/style.css">
  <script src="https://cdn.jsdelivr.net/npm/devtemdesign@0.0.2/script.js"></script>
</head>
<body>
  <div id="app-root"></div>
  <script>
    devtemDesign("app-root", {
      template: "cyber",
      logoName: "MyBrand",
      navLinks: ["Home", "Docs", "About"],
      heroTitleLine1: "Hello",
      heroTitleLine2: "World"
    });
  </script>
</body>
</html>

CDN Usage

Use jsDelivr for fast, reliable delivery. We recommend pinning a specific version for production.

<!-- Pinned version (stable) -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/devtemdesign@0.0.2/style.css">
<script src="https://cdn.jsdelivr.net/npm/devtemdesign@0.0.2/script.js"></script>

<!-- Always latest (use with caution) -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/devtemdesign/style.css">
<script src="https://cdn.jsdelivr.net/npm/devtemdesign/script.js"></script>

Font Awesome is required for icons (features, social links, etc.):

<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.4.0/css/all.min.css">

Features & Panels

Define features (grid cards) and panels (supplemental blocks). Both are arrays of objects.

features: [
  { 
    title: "Deterministic Schema", 
    desc: "Map JSON to hardware-accelerated layouts.", 
    icon: "<i class='fas fa-cube'></i>" 
  }
],
panels: [
  { tag: "API", title: "devtemDesign({...})", desc: "Single handler." }
]

Section labels are customizable:

  • featuresSectionLabel — text or HTML above the features grid
  • featuresSectionTitle — main heading for the features section

Footer Configuration (v0.0.2+)

The footer is fully declarative via a footer object. All fields are optional.

footer: {
  tagline: "Declarative UI engine for the modern web.",
  copy: "© 2025 Figsh Digital · MIT License",
  columns: [
    {
      heading: "Product",
      links: [
        { label: "Documentation", href: "/docs" },
        { label: "Sandbox", href: "/sandbox" }
      ]
    }
  ],
  social: [
    { icon: "fab fa-github", href: "https://github.com/devtem-dev", label: "GitHub" }
  ],
  legal: [
    { label: "Privacy Policy", href: "/privacy" },
    { label: "Terms", href: "/terms" }
  ]
}

Legacy The simple footerCopy string is still supported as a fallback.

Customization

All visual variables (accent colors, glass effects) are CSS custom properties. Override them after including the engine stylesheet:

<style>
  :root {
    --accent: #ff7b2c;
    --accent2: #ff2c7b;
    --glass-bg: rgba(20, 20, 40, 0.5);
  }
</style>

Browser Support

Modern browsers: Chrome, Firefox, Safari, Edge (latest 2 versions). Internet Explorer is not supported due to backdrop-filter and CSS Grid requirements.

Contributing & Repository

devtemDesign is open source on GitHub: github.com/devtem-dev/devtemDesign

We welcome contributions! Fork the repo, create a branch, and submit a pull request. Ideas:

  • New templates (pastel, dark-mono, etc.)
  • Extra config options (enableParallax, marqueeSpeed)
  • Accessibility improvements (ARIA labels, focus management)
  • Mobile touch gesture enhancements
